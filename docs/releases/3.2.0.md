# ArkGate 3.2.0

- **Released:** 2026-07-15
- **npm:** `arkgate@3.2.0` (`latest`)
- **Git tag:** `v3.2.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.2.0
- **Status:** published

ArkGate 3.2 adds contract health (Phase W): the same engine that validates code against the
contract now also meta-lints the contract itself and describes its governance weight, and the
docs name the advisory-local / hard-CI boundary as a deliberate trade-off. Every new surface is
**advisory only** — no verdict changes. **No breaking** CLI or `ark.config.json` changes.
**No gate weaken.**

## Highlights

| Area | 3.2.0 result |
|------|---------------|
| **Contract smells (W01)** | `doctor.contractHealth` lints the contract itself: explicitly bidirectional allows, peripheral-into-core allows, lateral adapter-to-adapter allows, and dead rules (unknown/empty layers, same-layer no-ops) |
| **Acknowledgments** | Deliberate edges live in a bounded `.ark/contract-smell-acks.json` sidecar — the versioned config schema is untouched; malformed files or edge grammar never suppress a smell |
| **Governance weight (W02)** | Raw facts (layers, rules, governed files, files/layer, rules/layer) plus a fixed band `heavy / typical / light / unknown` with `notAScore: true` — never a gate input |
| **Positioning honesty (W03)** | README/docs explain why the hard guarantee lives at the required merge status, and frame the contract as a pressure sensor for when the design stops fitting |
| **Hostile-input hardening** | FIFO/symlink/oversized ack files cannot hang or exhaust `--doctor`; banding is size-relative and NaN-safe; meta-lint follows manifest-effective rules |

## Try the release

```bash
npm install -D arkgate@3.2.0
npx ark-check --doctor --json   # doctor.contractHealth: smells + governanceWeight
npx ark-check --doctor          # human "Contract health (advisory)" section
```

Acknowledge a deliberate edge (example):

```json
// .ark/contract-smell-acks.json
{
  "acks": [
    {
      "id": "contract-bidirectional-allow",
      "edge": "WorkflowSagaEngine<->ApplicationOrchestration",
      "reason": "saga callbacks are a deliberate loop; cycles guarded by review"
    }
  ]
}
```

The MCP registry descriptor is version-aligned in `server.json`; registry publication remains a
separate maintainer step after npm is visible on `latest`.

## Compatibility and honesty

- All 3.2.0 surfaces are additive within ArkGate 3.x and advisory only: nothing in
  `contractHealth` changes a pass/fail verdict, `designFitness`, `patternBets`,
  `postGreenPath`, or any exit code.
- Layer roles for contract smells come from layer-NAME heuristics; a misread costs a warning
  line, never a verdict. Governance-weight bands use fixed documented thresholds and an explicit
  `notAScore: true` — no composite score, ranking, or gate input exists.
- Absent sidecar and absent smells are normal; `contractHealth.acknowledged` counts applied
  acknowledgments only (stale entries count zero).
- MCP registration is advisory unless the host makes that path non-bypassable; a hard local write
  claim remains operation-scoped; required CI status remains unverified locally. Behavioral
  completion is not evaluated by any of these surfaces.
