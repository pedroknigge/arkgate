# ArkGate 3.8.2

- **Released:** 2026-07-22
- **npm:** `arkgate@3.8.2` (`latest`)
- **Git tag:** `v3.8.2`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.8.2
- **Status:** published

ArkGate 3.8.2 is a backward-compatible corrective **patch** over 3.8.1 driven by PREDIAL WEB field
evidence. It aligns doctor skill freshness with managed-upgrade content identity, stops urging a
phantom `--apply` when content already matches, clarifies doctor writePath inventory vs this
invocation, ships the Y06 pure-layer opt-in advisory, and treats leftover Codex flat prompts as
safe-to-delete when the modern catalog is complete.

## What changed

| Area | 3.8.2 result |
|------|--------------|
| Doctor skill stale | Content identity (stamp-normalized) matches `ark upgrade`; lagging `arkVersion` alone is not stale |
| Upgrade preview | `wouldWrite` / customized preserved; “Nothing to apply” when content matches package |
| writePath / enforcement | `sessionNote` for unknown active host; package `installed` independent of host support; hard still needs runtime proof |
| Y06 | Advisory pure-layer opt-in when golden names pure modules and no layer sets `pure: true` |
| Codex legacy prompts | Complete `.agents/skills` + leftover prompts → advisory safe-to-delete (not install gap) |

## Upgrade

```bash
npm install -D arkgate@3.8.2
npx arkgate upgrade
```

Existing projects do not need a config migration. Skills whose content already matches package
templates should report current; doctor should no longer list stamp-only lag as content-behind.

**No required config migration.**

## Compatibility and limits

- No breaking CLI command, MCP tool, or required `ark.config.json` field change.
- Hard write enforcement still requires runtime observation for the active host (Z10).
- Y06 is advisory only and never auto-writes `pure: true`.
- Z09 and residual `RB-11` remain open for retained adoption and independently signed close.
