# ArkGate 4.0.0

- **Prepared:** 2026-07-24
- **npm:** `arkgate@4.0.0` (publish pending — runbook below)
- **Git tag (when publishing):** `v4.0.0`
- **Status:** prepared (not yet published to npm `latest`)

**Major** over **3.9.2**. Breaking removal of deprecated root runtime/nestjs forwarders plus
**opt-in ArkRules**: intra-layer structural sensors, invariant catalogs, coverage, and brownfield
rules inventory on the same enforcement plane.

Last published 3.x remains **`arkgate@3.9.2`** until this prepare is published.

## Ship window honesty

| Included in package | Claim boundary |
|---------------------|----------------|
| AR01–AR08 foundations + sensors + templates | Primary 4.0.0 product claim |
| AR04 root forwarder removal | **Breaking** — required migration for `arkgate/runtime` importers |
| AR09–AR19 surfaces in tree (invariants, inventory, doctor rules section, skills deepen) | Shipped code, progressive/opt-in; **not** “field train closed” |
| Z09 / RB-11 retained adoption | **Still open** (claim gate, not engineering) |

## Breaking

Root package subpaths **removed**:

- `arkgate/runtime`
- `arkgate/nestjs`

Use `@arkgate/runtime` and `@arkgate/runtime/nestjs` instead. See
[migration note](../migrate-from-ark-runtime-kernel.md).

## What changed (summary)

| Area | 4.0.0 result |
|------|----------------|
| Config | Optional `arkRules` map; schema `1.0→1.1` additive |
| Schemas | `arkgate/schema/arkrules`; analysis-result **1.4** provenance |
| Sensors | Closed Tier-1/2 vocabulary; classShapes on facts 1.1; fileHints |
| Empty scope | `ARKRULE_SCOPE_EMPTY` for zero-match `appliesTo` |
| Invariants | Catalog + test-title/symbol coverage; promotion refuse without evidence |
| Brownfield | `--rules-inventory` / MCP; freeze residual keys; extraction cards |
| Opt-in | No `arkRules` → same inter-layer verdict as 3.9.x |

## What did not change

- Binary gate; no numeric score; no LLM pass/fail
- No new skill *names* (deepen only)
- Z09 / RB-11 field claims remain open
- Experimental `@arkgate/runtime` stays experimental under ADR 0004
- Statement / function / line coverage floors and mutation gates (branch floor only: **84.5% → 83.0%**)

## Known residual (honest, non-blocking for prepare)

| Residual | Severity | Status |

## Upgrade

```bash
npm install -D arkgate@4.0.0
# if you imported arkgate/runtime or arkgate/nestjs:
#   change to @arkgate/runtime or @arkgate/runtime/nestjs
npx arkgate-check --install-agent-gates --force
npx arkgate-check --doctor --json
# optional ArkRules:
#   add "arkRules": { "DomainModel": "arkrules/DomainModel.json" } and starter files
```

## Publish runbook (maintainers)

1. CI green on this tag candidate  
2. `npm run build && npm run check:package-files && npm run test:package-isolation`  
3. Signed tag `v4.0.0` + GitHub Release from this file  
4. `gh workflow run publish-npm.yml -f tag=v4.0.0 -f dry_run=false`  
5. Verify: `npm view arkgate@4.0.0 version` and update this note **Status: published**  
6. `mcp-publisher validate server.json && mcp-publisher publish server.json`
