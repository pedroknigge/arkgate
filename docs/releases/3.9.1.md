# ArkGate 3.9.1

- **Prepared:** 2026-07-23
- **npm:** `arkgate@3.9.1` (not yet on `latest` until publication checklist succeeds)
- **Target tag:** `v3.9.1`
- **GitHub Release:** create at publish time → `https://github.com/pedroknigge/arkgate/releases/tag/v3.9.1`
- **Status:** prepared (registry `latest` remains **3.9.0** until this patch publishes)

Patch over **3.9.0**. Fixes CI/onboarding honesty after 3.9.0 reached npm, Dependabot hygiene for
eval Next fixtures and transitive `fast-uri`, and release-surface docs. No required config migration.
Does not weaken write gate or CI. Does not claim consumer CVEs from eval fixtures.

## What changed

| Area | 3.9.1 result |

| Onboarding tests | `o04` allows package-manager lockfile rewrite under `start --install` |
| Docs | Three-lane IA retained; published-wording path for post-npm-flip |

| Eval Next | **15.5.21** in `eval/cases/next-core-imports-db` and monorepo frontend fixture |
| Product surface | Unchanged from 3.9.0 (Beautiful Path + hosts) |

## What did not change

- Write gate / CI fail-closed semantics
- Host hardness claims (Claude/Grok/Antigravity hard when covered; Cursor/Codex/OpenCode advisory)
- Z09 / RB-11 field claims (still open)
- Analysis engine / plan-B policy

## Upgrade

```bash
npm install -D arkgate@3.9.1
# or: npm install -D arkgate@latest   # after this patch is latest
npx arkgate-check --install-agent-gates --force
npx arkgate-check --doctor
```

No config migration. Optional: refresh gates/skills after upgrade.

## Maintainer publication checklist

1. CI green on the release SHA  
2. `npm run release:npm -- --dry`  
3. Signed tag `v3.9.1` + GitHub Release from this file  
4. `gh workflow run publish-npm.yml -f tag=v3.9.1 -f dry_run=false`  
5. Verify: `npm view arkgate@3.9.1 version` and `npm view arkgate dist-tags.latest` → `3.9.1`  
6. Flip CONTRIBUTING / README / this status to **published**  
7. Optional: `mcp-publisher publish server.json` (requires valid registry JWT)
