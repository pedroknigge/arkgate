# ArkGate 3.0.5

**Released:** 2026-07-14  
**npm:** `arkgate@3.0.5` (`latest`)  
**Git tag:** `v3.0.5`  
**GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.0.5  
**Status:** published

Patch on stable 3.0: **Codex skill catalog** + **host honesty** so AGENTS.md `/ark-*`
commands match what Codex can load. **No breaking** CLI or `ark.config.json` changes.
**No gate weaken.**

## Highlights

| Area | 3.0.5 result |
|------|----------------|
| **Codex skills** | Repo: `.agents/skills/<name>/SKILL.md`; home: `$CODEX_HOME/skills/<name>/SKILL.md` |
| **Catalog verify** | Install fails if AGENTS.md `/ark-*` refs are missing from selected host catalogs |
| **Parity / legacy** | Doctor reports missing, stale, and legacy-prompts-only (flat prompts are not skills) |
| **CI fail-closed** | Gap when CI runs ark-check without `--strict` / `--strict-merge` / `--require-gates` |
| **Write path** | Codex install/doctor: advisory local write; CI is the hard merge backstop |
| **Temp safety** | Temp project roots never rebind default `~/.codex` MCP with a disposable `--root` |

## Upgrade

```bash
npm install -D arkgate@3.0.5
npx arkgate-check --install-agent-gates --tools codex --skills-only --force
# optional multi-project home skills:
npx arkgate-check --install-agent-gates --skills-only --codex-home --force
npx arkgate-check --doctor
```

Restart Codex after home MCP/skills changes.
