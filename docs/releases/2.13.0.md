# ArkGate 2.13.0

**Prepared:** 2026-07-11
**npm:** `arkgate@2.13.0`
**Git tag:** `v2.13.0`

This release also establishes the public analysis IR/API (`loadContract`, `analyzeProject`,
`analyzeChange`, and `explainViolation`) for adapter migration in C03.
**Merged feature PRs:** #29 (Phase S), #30 (C01)

> **Release gate:** this version is prepared but must not be published to npm `latest` while
> `ROADMAP.md` release blocker `RB-06` is open. Close `O03`, rerun the release gates on `main`,
> then create the signed tag and GitHub Release. A merge alone does not publish this package.

## Highlights

| Area | What changes |
|------|--------------|
| **Versioned configuration** | `ark.config.json` now uses `$schema` and `schemaVersion: "1.0"`; one loader validates, migrates, and rejects unknown keys consistently across CLI, MCP, and ESLint. |
| **Published schema** | The generated JSON Schema ships at `arkgate/schema` and `arkgate/schema/ark.config.schema.json`. |
| **Truthful enforcement** | Doctor and adoption checks report hard write, advisory write, merge gate, and repair capabilities for the active host instead of borrowing evidence from another installed host. |
| **Host-compatible setup** | Generated CI uses `--strict-merge`; optional `--require-write-hook <host>` checks only capabilities the selected host can actually provide. |
| **Scanner hardening** | Shadowed globals stop false-positive matching, while ambient aliases, `globalThis`, TypeScript `import = require`, and non-literal `require()` bypasses are handled explicitly. |
| **Regression confidence** | CI and both npm publish paths run broad coverage plus Stryker mutation testing before release. |
| **Runtime retry safety** | A persistence or completion-audit failure after a successful workflow effect is terminal and cannot execute that effect again. |

The product name remains **ArkGate**, the npm package remains **`arkgate`**, and both the
`arkgate*` and `ark*` command families remain supported.

## Upgrade

After the release gate is closed and `2.13.0` is published:

```bash
npm install -D arkgate@2.13.0
npx arkgate-check --install-agent-gates --force
npx arkgate-check --doctor
```

- Existing unversioned configs migrate deterministically in memory.
- Persist `$schema` and `schemaVersion: "1.0"` when intentionally adopting the new contract.
- Review unknown-key diagnostics before removing fields; validation fails closed by design.

## Maintainer publish checklist

Run these steps only after `RB-06` is closed and the release-preparation PR is merged to `main`:

```bash
# 1. Update main and confirm the release blocker is closed
git checkout main
git pull --ff-only
rg 'RB-06.*closed' ROADMAP.md
rg 'O03.*done' ROADMAP.md

# 2. Confirm the four version sources agree
npm test -- --run tests/unit/smoke.test.ts
# Expect package.json, package-lock.json, src/version.ts, and server.json at 2.13.0

# 3. Run the complete release verification and package dry run
npm run release:npm -- --dry

# 4. Create and push a signed annotated tag
git tag -s v2.13.0 -m "arkgate v2.13.0"
git push origin v2.13.0

# 5. Create the required GitHub Release
gh release create v2.13.0 --verify-tag \
  --title "arkgate v2.13.0" \
  --notes-file docs/releases/2.13.0.md

# 6. Explicitly publish with provenance; this workflow never runs on merge
gh workflow run publish-npm.yml -f tag=v2.13.0 -f dry_run=false

# 7. Verify the public artifacts
npm view arkgate@2.13.0 version
gh release view v2.13.0
```

Emergency local publishing remains outside the normal release path and is not part of this
preparation.
