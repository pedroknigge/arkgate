# ArkGate 2.12.0

**Date:** 2026-07-10  
**npm:** `arkgate@2.12.0`  
**Git tag:** `v2.12.0`  
**PR:** #26 (`feat/fix-explore-gaps`)

## Highlights (user-facing)

| Area | What you get |
|------|----------------|
| **Day-zero origin** | `ark start` / `ark init` freeze `.ark/reports/origin.*` as soon as the contract exists, **before** agent docs and CI templates. Later `--report` shows evolution. |
| **Explore / autopilot** | Decision-grade recon; dual plan **A** (remediation) + **B** (pattern bets; never auto-applied as mechanical-safe). Empty plan is not “healthy” without explore. |
| **Write path dogfood** | This repo’s Claude/Grok hooks use `--hook-repair`; doctor `writePath.mode = repair`. |
| **Install honesty** | Temp/upgrade scratch roots no longer rewrite the developer’s real `~/.codex/config.toml` unless `CODEX_HOME` is set. |
| **CI Node default** | Generated ark-check workflow defaults to Node **24** when the project declares none (and inherits sibling workflow majors when present). |

## Maintainer publish checklist

Use after PR #26 is merged to `main` and CI is green on the merge commit:

```bash
# 1. On main at the release SHA
git checkout main && git pull

# 2. Confirm versions agree
node -e "const p=require('./package.json');const s=require('./server.json');const v=require('./src/version.ts');" 2>/dev/null || true
grep -E "version|2\.12\.0" package.json src/version.ts server.json | head -10
# Expect package.json, src/version.ts, server.json all 2.12.0

# 3. Dry-run gates + pack
npm run release:npm -- --dry

# 4. Signed annotated tag (GPG key required)
git tag -s v2.12.0 -m "arkgate v2.12.0"
git push origin v2.12.0

# 5. GitHub Release (required before publish workflow)
gh release create v2.12.0 --verify-tag \
  --title "arkgate v2.12.0" \
  --notes-file docs/releases/2.12.0.md

# 6. Publish to npm with provenance
gh workflow run publish-npm.yml -f tag=v2.12.0 -f dry_run=false

# 7. Verify
npm view arkgate@2.12.0 version
gh release view v2.12.0
```

Emergency local publish (no provenance): `npm run release:npm -- --allow-local` — not the default path.
