# ArkGate 3.4.0

- **Released:** 2026-07-16
- **npm:** `arkgate@3.4.0` (`latest`)
- **Git tag:** `v3.4.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.4.0
- **Status:** published

ArkGate 3.4 completes Phase U (understandable execution): the typed capability evidence from 3.3
becomes **opt-in enforcement** on every surface, an advisory ambient-state sensor lands for pure
layers, and the real pre-tool path is measured before any threshold exists. Everything is
opt-in — a config without the new keys behaves exactly as before. **No breaking** CLI or
`ark.config.json` changes. **No gate weaken.**

## Highlights

| Area | 3.4.0 result |
|------|---------------|
| **Capability walls (U04)** | Per-layer `capabilities: { deny: [...] }` or `pure: true` (deny all seven). Judgment-class `CAPABILITY_VIOLATION` with a port `nextAction` — never mechanical-safe, never auto-patched |
| **Every adapter** | CLI scan (ambient + import), pure IR engine, atomic multi-file preflight, the real PreToolUse hook / MCP gate, and ESLint (`ark/no-denied-capabilities`) enforce the same deny set |
| **One voice (D7)** | An ambient use covered by the layer's `forbiddenGlobals` reports only `FORBIDDEN_GLOBAL` |
| **Coverage-atom delta (D6)** | Policy transitions classify on coverage atoms: any lost atom (`fetch`→`XMLHttpRequest`, `Date`→`Date.now`, wall→fg) requires the hash-bound acknowledgment; fg → equivalent-or-stronger wall never does |
| **Ambient state (U05)** | `doctor.ambientState` flags module-scope `let`/`var` in `pure: true` layers — advisory, sidecar acks, no strict mode anywhere |
| **Measured pre-tool path (U06)** | `bench:hook-path` measures the complete hook/doctor child processes; ceilings land only from the recorded Linux baseline (D5 method) |

## Try the release

```bash
npm install -D arkgate@3.4.0
```

Declare a pure domain (casual surface):

```json
{
  "layers": [
    { "name": "DomainModel", "patterns": ["src/domain/**"], "pure": true }
  ]
}
```

Or deny specific effects (expert surface):

```json
{ "name": "DomainModel", "patterns": ["src/domain/**"], "capabilities": { "deny": ["clock", "network", "persistence"] } }
```

Every surface now blocks a denied effect with the same dual-depth answer: a plain hint
("define a ClockPort / HttpPort / StoragePort and bind it in an adapter layer") plus stable JSON
(`ruleId`, `capability`, `fixClass: inject-port`, deterministic `nextAction`).

## Compatibility and honesty

- All 3.4.0 surfaces are opt-in and additive: absence of `capabilities`/`pure` changes no
  verdict; brownfield projects gain no surprise blockers.
- The pure engine, preflight, and ESLint enforce the **import** dimension (exact module/subpath
  matching, never substring; `import type`/`export type` erased; template-literal text skipped;
  `require()` counts as evidence, never as a graph edge). Ambient globals are enforced on the
  symbol-aware CLI/hook/MCP paths (shadowing, aliasing, `globalThis` precision).
- `CAPABILITY_VIOLATION` is judgment-class: the write gate proposes a port; it never rewrites
  code silently, and autoPatch never patches a wall violation away.
- The ambient-state sensor is advisory and opt-in; no strict mode exists, and none may be added
  without a completed false-positive corpus (ADR 0009 A5).
- Hook-path budgets follow the D5 method: recording mode until the Linux CI baseline exists;
  ceilings are baseline + fixed headroom, set once per cycle, never ratcheted.
- MCP registration is advisory unless the host makes that path non-bypassable; hard local write
  claims remain operation-scoped; required CI status remains unverified locally.
