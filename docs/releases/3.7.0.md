# ArkGate 3.7.0

- **Released:** 2026-07-17
- **npm:** `arkgate@3.7.0` (`latest`)
- **Git tag:** `v3.7.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.7.0
- **Status:** published

ArkGate 3.7.0 closes the promoted Phase Y field-feedback work: reshape choices become explicit
project records, design and parser blind spots become visible without changing the verdict,
skill instructions gain explicit mechanical-edit hygiene requirements backed by deterministic
fixture checks, and the confirmed `node:process` purity escape is closed across every adapter.

## What changed

| Area | 3.7.0 result |
|------|--------------|
| Reshape decisions (Y01) | Optional `.ark/reshape-decisions.json` records `accepted`, `deferred`, or `rejected` verdicts with reasons and lifecycle; facts remain visible and invalid/stale records suppress nothing |
| Hollow persistence (Y02) | `handler-in-persistence` recognizes bounded static HTTP-import, route-definition, and handler evidence in Persistence-role modules; advisory only |
| Parse honesty (Y03) | `doctor.parseHealth` and the HTML report show parser-diagnostic totals plus a deterministic capped file list from the existing AST; verdict unchanged |
| Mechanical edits (Y04) | Ark skill instructions require preserving existing doc blocks and typed route calls and avoiding placeholder files; a deterministic eval validates accepted/rejected fixture shapes |
| Budgets (Y05) | Package and Linux hook/doctor ceilings are fixed from clean 3.6.1 measurements with explicit headroom |
| Confidence budget | Meaningful Y08 listener regressions are added; the full-suite branch floor is explicitly recalibrated from 85% to 84.5%, with other coverage floors and mutation unchanged |
| `process` module dual (Y08) | Exact value imports of `process` / `node:process` are one `FORBIDDEN_GLOBAL` voice when the layer declares `forbiddenGlobals: ["process"]`; subpaths, `child_process`, and documented type-only forms remain excluded |
| Focused fixes | Peer slice identity is case-portable, and convergence now asks to restore an unexpectedly removed dependency |

## Upgrade

```bash
npm install -D arkgate@3.7.0
npx arkgate upgrade
```

Existing projects do not need a config migration. To record an explicit reshape decision, use the
exact `decisionTarget` emitted by the proposed pilot; Ark's skills document the bounded sidecar
shape and keep the decision behind the normal write gate.

## Compatibility and safety

- No breaking CLI command, MCP tool, analysis IR, or `ark.config.json` schema change.
- Y01–Y03 remain additive/advisory: no decision, design smell, or parse-health count changes
  `ok`, the architecture verdict, or the process exit code. No reshape is applied automatically.
- Y08 intentionally strengthens an existing declared policy: a layer that forbids ambient
  `process` also forbids its two exact runtime module spellings. The broader `process` capability,
  subpaths, and `child_process` are not inferred from that declaration.
- Peer isolation uses portable case-normalized slice identity. Paths differing only by case are
  not treated as distinct slices because that distinction cannot survive macOS/Windows clones.
- The release confidence run covers 6,661 / 7,854 branches (84.81%) against the explicit 84.5%
  branch floor. No coverage include/exclusion changed, and mutation remains mandatory.
- Y06, Y07, Y09, and Y10 remain parked until their field-evidence gates are met; they are not
  included in this release.
