# ArkGate 3.8.0

- **Released:** 2026-07-21
- **npm:** `arkgate@3.8.0` (`latest`)
- **Git tag:** `v3.8.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.8.0
- **Status:** published

ArkGate 3.8.0 is a backward-compatible corrective minor. It makes incomplete analysis fail
closed, restores one resolved architecture verdict across parity-capable adapters, hardens clean
installation and managed upgrades, and adds an opt-in base-relative design-delta gate.

## What changed

| Area | 3.8.0 result |
|------|--------------|
| Analysis completeness (Z02) | CLI, MCP, hook, and public results carry required `complete`, `partial`, or `unavailable` evidence; incomplete results cannot satisfy plans or strict merge |
| Resolved verdict parity (Z04) | Complete create/update/delete candidates are resolved once into versioned facts used by the API, CLI, MCP, complete-patch hook, and final check |
| TypeScript distribution | A physically distinct exact TypeScript 6 analysis host prevents a consumer TypeScript 7 install from replacing ArkGate's JS API fallback |
| Managed upgrades (Z06) | Upgrade preview and apply are separate confirmations bound by a SHA-256 plan digest; managed content is classified without touching unrelated files |
| Enforcement evidence (Z06/Z10) | Schema 1.1 requires fresh runtime and operation evidence before a host path can report `hard:true` |
| Warm analysis (Z07) | Identity-keyed resident analysis reuses valid compiler work without trusting legacy cache snapshots |
| Causal evidence (Z08) | Preregistered comparison evidence replaces broad speed claims with measured, bounded results |
| Design delta (Z10) | `--fail-on-new-smells --base-ref <ref>` blocks only new or worsened `domain-logic-in-ui` evidence and fails closed when its base cannot be resolved |
| Release safety (Z01) | Dry-run cleanup and packed-artifact checks are bounded so verification cannot remove unrelated paths |

## Upgrade

```bash
npm install -D arkgate@3.8.0
npx arkgate upgrade
```

Existing projects do not need a config migration. `ark upgrade` first produces a read-only managed
content preview; applying managed-file changes requires the exact `planDigest` from that preview.

## Compatibility and limits

- No breaking CLI command, MCP tool, or required `ark.config.json` field change.
- New result fields, schemas, and CLI flags are additive, making this a minor release.
- Lexical and single-snippet compatibility paths remain explicitly partial and non-green; strict
  CI is the final merge boundary.
- Design-delta enforcement is opt-in and base-relative. Existing historical smells do not block
  unless they are new or worsened in the candidate.
- `@arkgate/runtime` is experimental, separately versioned, and not published by this release.
- Z09 and `RB-11` remain open for retained adoption and independently signed close. This release
  does not claim that Phase Z is shipped or that its broad product claims are earned.
