# ArkGate 3.8.3

- **Released:** 2026-07-22
- **npm:** `arkgate@3.8.3` (`latest`)
- **Git tag:** `v3.8.3`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.8.3
- **Status:** publishing

ArkGate 3.8.3 is a backward-compatible corrective **patch** over 3.8.2 driven by multi-repo field
evidence (PROPIA pnpm workspace upgrade failure; Amarilla greenfield `start` incomplete). It fixes
package-manager install argv for workspace roots, pins `arkgate` on `start --apply` by default,
always writes project `.mcp.json` in compact setup, and hardens upgrade recovery/UX.

## What changed

| Area | 3.8.3 result |
|------|--------------|
| pnpm workspace install | `pnpm add -D arkgate@… -w` at workspace roots |
| yarn workspaces | `yarn add -D … -W` when `package.json` workspaces declared |
| Already-current package | Skip install; recompute managed preview |
| `start --apply` | Pins `arkgate` by default; `--no-install` opt-out; runs PM after apply |
| Compact gates | Always write `.mcp.json`; budget 8 files / 32 KB |
| Upgrade unbound apply | No-op success when content matches (`wouldWrite=0`) |
| Upgrade applied copy | Stamp refresh vs content write counts |

## Upgrade

```bash
# npm
npm install -D arkgate@3.8.3
npx arkgate upgrade

# pnpm monorepo root (3.8.3+ emits -w itself)
pnpm add -D arkgate@3.8.3 -w
pnpm exec ark upgrade --apply
```

**No required config migration.**

## Compatibility and limits

- Compact start still omits full `/ark-*` skill catalogs (by design); use
  `ark-check --install-agent-gates --skills-only --force` for full skills.
- Hard write enforcement still requires runtime observation for the active host (Z10).
- Z09 and residual `RB-11` remain open for retained adoption and independently signed close.
