# ArkGate 3.0.4

**Released:** 2026-07-14  
**npm:** `arkgate@3.0.4` (`latest`)  
**Git tag:** `v3.0.4`  
**GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.0.4  
**Status:** published

Patch on stable 3.0: **report honesty** + **showcase depth** so HTML/`latest.json` match
doctor intent. **No breaking** CLI or `ark.config.json` changes. **No gate weaken.**

## Highlights

| Area | 3.0.4 result |
|------|----------------|
| **Mode badge** | HTML report uses the same four **core** layers as doctor for optional-core ADAPT; secondary optional layers no longer force false **ADAPT** |
| **Adoption / write-path** | `write-path-none` not opened when session host is `unknown` but inventory has hard/advisory gates; `mode` stays honest for the process |
| **Grok detect** | `GROK_AGENT` recognized by `detectActiveAgentHost` |
| **KPI hints** | Every hero / adoption / density / debt KPI has plain-language micro-copy + tooltip |
| **Design-depth strip** | `--report` shows ENFORCE · design-weak, smell outcomes, one next pilot, post-green door (doctor parity) |
| **Adoption extras** | Write-path line (host · mode · inventory) + baseline policy legend |

## Honesty (read this)

- **PASS / ENFORCE** remain edge honesty. Design residual is **design-weak**, not FAIL.
- Design-depth strip requires a real sensor run; missing `designDepth` does **not** claim “Design depth · OK”.
- Write-path `mode: none` with host `unknown` can still show inventory gates on disk without a false adoption gap.

## Upgrade

```bash
npm install -D arkgate@3.0.4
npx arkgate-check --install-agent-gates --force   # optional skill/hook refresh
npx arkgate-check --root . --config ark.config.json --report ark-report.html
npx arkgate-check --doctor --json                 # designFitness / pilotLoop unchanged from 3.0.3
```

Prefer regenerating the report from an agent session (or `ARK_ACTIVE_HOST`) for session-accurate write-path **mode**; inventory and gaps are honest either way.
