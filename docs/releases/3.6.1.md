# ArkGate 3.6.1

- **Released:** 2026-07-17
- **npm:** `arkgate@3.6.1` (`latest`)
- **Git tag:** `v3.6.1`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.6.1
- **Status:** published

ArkGate 3.6.1 fixes Codex multi-project MCP resolution. Codex now gets a project-scoped
`.codex/config.toml`, so opening repository B no longer leaves `ark://manifest` bound to
repository A through the global home configuration.

## What changed

| Area | 3.6.1 result |
|------|--------------|
| Codex install | `--install-agent-gates --tools codex` writes `.codex/config.toml` with `--root .` and no longer mutates the global MCP registry |
| Doctor | Recognizes the effective project MCP and suppresses unrelated global multi-project warnings |
| Worktrees | Claude, Codex, and Grok worktree roots are classified as ephemeral, so stale agent worktrees cannot become permanent MCP owners |
| Compatibility | `--codex-home` retains the absolute-path global fallback for older clients or explicit home-wide registration |

## Upgrade

```bash
npm install -D arkgate@3.6.1
npx arkgate upgrade
```

Restart Codex after the upgrade. The generated project binding is:

```toml
[mcp_servers.ark]
command = "npx"
args = ["arkgate-mcp", "--root", ".", "--config", "ark.config.json"]
```

## Compatibility and safety

- No CLI JSON, MCP tool, or `ark.config.json` contract changed.
- Existing `$CODEX_HOME/config.toml` entries are preserved unless the maintainer explicitly
  invokes `--codex-home`; project configuration takes precedence when present.
- Ark's architecture rules and enforcement thresholds are unchanged.
