# ArkGate 3.9.0

- **Released:** 2026-07-23
- **npm:** `arkgate@3.9.0` (`dist-tags.latest`)
- **Git tag:** `v3.9.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.9.0
- **Status:** published

ArkGate 3.9.0 is a backward-compatible **minor** over 3.8.3. It ships **Beautiful Path** plus
residual product-beauty, write-path honesty, Shape coach depth, and field-claim scaffolding: one
primary adoption flow, doctor as the control plane, progressive disclosure (compact router first;
full skill pack is expert depth), and a canonical product-voice document so public English stays
senior-grade and honest. It does **not** rewrite the analysis engine, weaken the write gate, auto-
apply plan **B**, claim hard write on Cursor/Codex, or claim Phase Z field closure.

## What changed

| Area | 3.9.0 result |
|------|--------------|
| Product voice | `docs/product-voice.md` — lexicon (design-weak / residual / hard vs advisory), **Do** table, microcopy models, progressive disclosure |
| First-run narrative | README: `start` → doctor → optional guided autopilot; expert skills as escapes |
| Compact router | Primary path only; expert skills behind install + doctor/STOP |
| Doctor human output | Status lights + **Primary next action**; design-weak uses warn `!`; edge-clean under residual is not absolute “matches the contract”; New-here = finish start |
| Host write honesty | README/AI-gates/doctor: Claude/Grok/Antigravity hard PreToolUse when covered; Cursor/Codex/OpenCode advisory; **required CI status** is merge hard boundary |
| Host expansion | Antigravity (`.agents/hooks.json` + hook adapter) and OpenCode (`opencode.json` MCP; experimental plugin template only) |
| Skills | Autopilot/explore mode tables rank **Enforce · design-weak** Shape door; smell envelope honesty |
| Shape coach | Single door: explore shape-focus → dual-plan B → autopilot with OK; no silent codemod |
| Field kit | `docs/field/` scaffolding for Z09 (preregistration, cohort, independent reviewer) — **status: not closed** |
| Agent guide | Default path matches README; alternate tracks open with start → doctor |
| Docs IA | Three-lane hub: [docs/use.md](../use.md) · [docs/develop.md](../develop.md) · [CONTRIBUTING](../../CONTRIBUTING.md); history under [docs/archive/](../archive/README.md) |
| Maintainer audit | `.grok/workflows/product-beauty-audit.rhai` (read-only report) |

## What did not change

- Write gate / CI fail-closed behavior and strict-merge semantics.
- Mechanical-safe vs judgment split; plan B is never silent codemod.
- Post-green JSON ids: `postGreenPath`, `primaryNextAction`, `clarify-for-ai`.
- Skill template files remain shipped; nothing was deleted.
- Z09 / residual `RB-11` (retained adoption + independent close) still open — field kit is not close.

## Upgrade

```bash
# npm
npm install -D arkgate@3.9.0
npx arkgate upgrade

# pnpm monorepo root
pnpm add -D arkgate@3.9.0 -w
pnpm exec ark upgrade --apply
```

**No required config migration.**

Optional: refresh skill bodies after upgrade:

```bash
npx arkgate-check --install-agent-gates --skills-only --force
```

## First-run (the beautiful path)

```bash
npx arkgate start
npx arkgate start --apply
npx arkgate-check --doctor          # control plane
# optional guided work:
npx arkgate-check --install-agent-gates --skills-only --force
# then /ark-autopilot in the agent host
```

## Compatibility and limits

- Compact start still omits full `/ark-*` skill catalogs (by design).
- Hard write enforcement still requires runtime observation for the active host (Z10 evidence ladder).
- Design smell detectors use bounded envelopes (e.g. first 800 Persistence candidates); absence of a
  smell is not full-tree proof.
- Z09 and residual `RB-11` remain open for retained adoption and independently signed close.
- This release does not claim “design finished” for design-weak trees; Enforce remains edges-only.
