# ArkGate 3.3.0

- **Released:** 2026-07-16
- **npm:** `arkgate@3.3.0` (`latest`)
- **Git tag:** `v3.3.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.3.0
- **Status:** published

ArkGate 3.3 ships the first slice of Phase U (understandable execution): typed effect
capabilities as **evidence-only** architecture facts, the accepted ADR 0009 boundary, and a
legibility dogfood of the engine itself. Nothing blocks on capabilities in this release —
opt-in capability walls arrive with the second slice (3.4) after the evidence matures in the
field. **No breaking** CLI or `ark.config.json` changes. **No gate weaken.**

## Highlights

| Area | 3.3.0 result |
|------|---------------|
| **Capability evidence (U03)** | Seven closed ids (`network`, `filesystem`, `clock`, `randomness`, `environment`, `process`, `persistence`); public `analyzeProject(...).ir.capabilityUses` carries import-based evidence from the pure engine, while CLI/hook adapters use the symbol-aware collector internally (shadowing, type-only, and `globalThis` aliases handled) |
| **Boundary locked (U01/ADR 0009)** | Direct evidence blocks (later, opt-in); transitive inference never does; effects are architecture facts, not a style score — no inlining rules, no LOC caps, no trust score |
| **Executable corpus** | 25 frozen fixture cases (positives, adversarial `globalThis` alias, shadowing/type-only/similar-name negatives, a policy-allowed adapter case, and the D6 lowered-policy pair) run green against the shipped collector |
| **Engine legibility (U02)** | The canonical analysis entry is a pure facade over six cohesive kernel modules; generated config artifacts stayed byte-identical; ArkGate's own doctor reports zero design smells on this repository |
| **Runtime artifact** | The `@arkgate/runtime` source package builds a minified experimental artifact with stable names (`keepNames`), inside its artifact budget; it is not published by the root release workflow |

## Try the release

```bash
npm install -D arkgate@3.3.0
npx ark-check --doctor --json   # doctor surfaces unchanged; capabilities are evidence-only
```

Library consumers can inspect the new evidence:

```ts
import { analyzeProject, loadContract } from 'arkgate';
const contract = loadContract(arkConfig);
const { ir } = analyzeProject({ contract, files });
console.log(ir.capabilityUses); // [{ file, symbol: 'pg', capability: 'persistence', evidence }]
```

## Compatibility and honesty

- Everything in this release is additive within IR `1.0` and **evidence-only**: no verdict,
  wall, gate result, `designFitness`, or exit code reads capability uses until a layer opts into
  a policy in the second slice (U04).
- Historical API correction: `CAPABILITY_IDS` and `collectCapabilityUses` were internal
  implementation exports, not root exports from `arkgate@3.3.0`. The supported public evidence
  contract was and remains `analyzeProject(...).ir.capabilityUses`.
- The pure IR engine reports **import-based** evidence only (exact module/subpath matching,
  never substring; `import type`/`export type` erased). Ambient globals require the symbol-aware
  collector; mixed named-binding type imports and template-literal contents are documented
  envelope of the compiler-free scan.
- A workspace package literally named like a known driver (`pg`, `redis`) classifies by name —
  compiler-free matching never consults resolution (documented in ADR 0009).
- MCP registration is advisory unless the host makes that path non-bypassable; hard local write
  claims remain operation-scoped; required CI status remains unverified locally.
