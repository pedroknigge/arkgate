# ArkGate 3.6.0

- **Released:** 2026-07-17
- **npm:** `arkgate@3.6.0` (`latest`)
- **Git tag:** `v3.6.0`
- **GitHub Release:** https://github.com/pedroknigge/arkgate/releases/tag/v3.6.0
- **Status:** published

ArkGate 3.6 closes Phase X: the doctor learns to see **physical shape** — domain concepts
exploded across mirrored directory trees, invisible to every edge-based surface — and agents get
a governed loop to execute reorganizations one pilot at a time. Plus three field warm-ups from
the 3.5.0 the field adopter validation. Everything remains **advisory**: no verdict, exit-code,
`designFitness`, or `patternBets` change; **no apply path exists anywhere**. **No breaking** CLI
or `ark.config.json` changes. **No gate weaken.**

## Highlights

| Area | 3.6.0 result |
|------|---------------|
| **Physical cohesion (X04)** | `doctor.physicalCohesion` reports concept clusters per anchor directory — concentration, not volume (dispersed `use-*` hooks never fire). Fixed corpus-calibrated thresholds; anchors under `app/`/`pages/` are `fixedByConvention`; `notAScore` |
| **Reshape pilot (X04)** | `reshapePilot.nextPilot`: one card at a time, **proposed, never applied** — smallest convention-free anchor, `moveSample`/`movesTotal`, `successSignal`, `killSwitch`, hard `doNot[]`. The consolidation target subtree is never re-proposed: the loop converges |
| **Governed execution** | Real moves run only via `/ark-loop` through the write gate + atomic preflight (a pilot that would break the contract is blocked with the `/ark-contract` remediation); merges are `/ark-architect` **merge cards** — domain modeling, never a codemod; `/ark-fix` never folds reshapes into a fix batch |
| **Stale acks (X05)** | `contractHealth.ackLifecycle.staleCount` / `stale[]` name ack entries matching no detected edge — fix or delete, surfaced even at zero visible smells |
| **Mid-name families (X06)** | `HoursPersistenceAdapters -> PersistenceInfrastructure` goes quiet (family token anywhere in the source); role words never become a family (`AdaptersCore` is nobody's base) |
| **Evidence overflow (X07)** | Report findings announce the 6-item evidence cap with an honest `(+N more)` marker |

## Validation before this release

- **Multi-repo harness:** fresh clones of five known repos (Next.js app-router, SPA monorepo, backend monorepo, library, and server-library shapes) — zero crashes, all report parity sections rendered, doctor
  204–621 ms, zero false positives (the harness itself caught and fixed a monorepo
  scaffold-segment defect before release).
- **End-to-end pilot loop:** executed on a fixture with a real import graph — pilot 1 green,
  the naive pilot 2 **blocked by the gate** with the correct remediation (the preflight
  protecting as designed), kill switch verified, judgment pilot 2 green, and the loop reached
  its convergence state.
- **Field corpus (the field adopter, 2,996 files):** the sensor reproduced the ADR 0010 calibration
  table exactly — 4 mirrored concepts; zero findings on this repository (self-hosting negative).

## Compatibility and honesty

- All 3.6.0 changes are additive and advisory: absence of exploded concepts changes nothing;
  sidecars and configs from 3.5.0 behave identically.
- Concept extraction is a name/path heuristic (same class as the layer-role regexes): a miss
  costs a warning line, never a verdict. Thresholds are fixed constants with recorded corpus
  provenance (ADR 0010 D3), not tunables.
- There is **no apply path**: the pilot proposes; `/ark-loop` executes through the same write
  gate any agent faces, and a reshape that would create violations is blocked, not forced.
- Merges are never mechanical: the field corpus showed zero structural clones among 123
  same-concept files — merging is domain modeling, so it ships only as judgment cards.
- MCP registration is advisory unless the host makes that path non-bypassable; hard local write
  claims remain operation-scoped; required CI status remains unverified locally.
