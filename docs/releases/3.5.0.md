# ArkGate 3.5.0

- **Released:** pending
- **npm:** `arkgate@3.5.0` (pending)
- **Git tag:** `v3.5.0` (pending)
- **GitHub Release:** pending
- **Status:** prepared

ArkGate 3.5 is the field-feedback release (Phase X, from the amarilla adoption session — 2,996
files, 12 layers, ENFORCE at 100%): the HTML report reaches parity with the doctor **and stays
there by an executable rule**, acknowledgments gain a lifecycle so migration acks cannot
fossilize, and the lateral-adapter smell recognizes family infrastructure. Everything remains
**advisory** — no verdict, `designFitness`, or gate behavior changes. **No breaking** CLI or
`ark.config.json` changes. **No gate weaken.**

## Highlights

| Area | 3.5.0 result |
|------|---------------|
| **Report parity (X01)** | `ark-check --report` renders every doctor advisory: contract health with acknowledgment honesty, governance weight, ambient state, and `pure` / `walls: …` badges in the layers table |
| **The standing rule** | Executable, not prose: `reportParity.test.ts` enumerates the doctor's advisory keys and fails CI when one lacks its `data-advisory="<key>"` report section — the report can never fall behind the product again |
| **Ack lifecycle (X02)** | Optional `reviewBy` (`YYYY-MM-DD`) per ack; past the date the ack stops applying and the smell returns annotated. Undated acks keep applying but are surfaced even when everything is suppressed — the exact fossilization case from the field |
| **Lifecycle honesty** | `contractHealth.ackLifecycle` reports `{ undated, malformed, expiredCount, expired[] }`; malformed dates never apply (fail-loud); a re-ack with a fresh date wins over a dead entry, and once any dated ack exists for an edge the dated entries govern — an undated leftover cannot resurrect an expired exception |
| **Family infra (X03)** | `contract-lateral-adapter-allow` no longer fires on `PaymentsAdapters -> PaymentsInfra` — an adapter into its own family base is the sanctioned direction; cross-family, non-infra siblings, and base → member still fire |

## Try the release

```bash
npm install -D arkgate@3.5.0
npx ark-check --root . --report ark-report.html
```

Give a deliberate exception a review date (`.ark/contract-smell-acks.json`):

```json
{
  "acks": [
    {
      "id": "contract-bidirectional-allow",
      "edge": "ApplicationOrchestration<->WorkflowSagaEngine",
      "reason": "saga callbacks during the Q3 migration",
      "reviewBy": "2026-10-01"
    }
  ]
}
```

After `2026-10-01` the ack stops applying and the smell returns with `(ack expired 2026-10-01)`
in its evidence — re-review the edge and re-ack with a new date, or fix the contract.

## Compatibility and honesty

- All 3.5.0 changes are additive and advisory: no verdict, exit code, `designFitness`, or
  `patternBets` behavior changes; sidecars without `reviewBy` behave exactly as in 3.4.0.
- The family-infrastructure carve-out is a name heuristic like the existing role regexes — a miss
  costs a warning line, never a verdict. The reverse edge (family base importing a member
  adapter) deliberately still fires.
- Date validation is strict round-trip (`2026-02-30` is malformed, not silently March 2);
  malformed dates and edges never suppress a smell.
- The report advisory renderer keeps its injected-escaper seam (no dependency cycle with
  `html-report.mjs`) and its module budget (200 LOC).
- MCP registration is advisory unless the host makes that path non-bypassable; hard local write
  claims remain operation-scoped; required CI status remains unverified locally.
