# ArkGate 3.0.0

**Prepared:** 2026-07-13  
**npm:** `arkgate@3.0.0`  
**Git tag:** `v3.0.0`  
**Status:** prepared; not tagged or published

ArkGate 3.0.0 is the first stable release prepared after the V05 independent beta-exit audit
passed in PR #49. The root package remains the architecture gate; `@arkgate/runtime` and the
NestJS surface remain optional experimental packages.

  97% median governed coverage, all four supported hosts, zero P0/P1 findings, and independent
  review recorded as `pedroknigge`.

  isolation, release artifacts, fuzz, and performance checks passed.
- CI: PR #49 passed build, CodeQL, Semgrep, onboarding, compatibility, deterministic fuzz,
  performance, and adapter-parity checks before merge to `main` as `6bf43ae`.
- Release artifacts: gate/runtime tarballs, SBOMs, checksums, manifests, size budgets, and

## Upgrade

```bash
npm install -D arkgate@3.0.0
npx arkgate-check --install-agent-gates --force
npx arkgate-check --doctor
```

The `arkgate*` and `ark*` CLI aliases remain supported. Existing `ark.config.json` files retain
their versioned schema and fail-closed validation behavior.

## Maintainer publication checklist

Run this only from a clean checkout of `main` and only after explicit publication authorization:

```bash
# 1. Verify the exact prepared candidate without publishing
npm run release:npm -- --dry

# 2. Create and push the signed release tag
git tag -s v3.0.0 -m "arkgate v3.0.0"
git push origin v3.0.0

# 3. Create the GitHub Release required by the publish workflow
gh release create v3.0.0 --verify-tag \
  --title "arkgate v3.0.0" \
  --notes-file docs/releases/3.0.0.md

# 4. Publish with GitHub Actions provenance
gh workflow run publish-npm.yml -f tag=v3.0.0 -f dry_run=false

# 5. Confirm public artifacts
npm view arkgate@3.0.0 version
gh release view v3.0.0
```

The normal release path never uses local publishing. `publish-npm.yml` verifies the signed tag,
GitHub Release, unpublished version, release artifacts, and npm provenance before publishing.
